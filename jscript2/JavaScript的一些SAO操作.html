    IE判断检测

    jQuery 在 1.9 版本之前，提供了一个浏览器对象检测的属性 使用率极高。但是在版本发布之后，大家钟爱的这个属性被无情的抛弃了。大家开始着手寻找.browser 的替代方案。于是各种利用 IE bug 的检测方法被搜了出来：

    // IE 678 最短方法 
    var ie = !-[1,];

    解析 : 
    在标准浏览器下 
    false

    IE 6 7 8 下: 
    true

    拆解步骤

    /* IE678 */ 
    [-1,] //[object Array] [-1, undefined] 逗号后位置初始化为undefined未省略 2 
    -[-1,] //NaN 
    /* 非IE */ 
    [-1,] // [-1] 省略了 , 之后空位 1 
    -[-1,] // 1 隐式计算 先转化为number然后进行 --取反
